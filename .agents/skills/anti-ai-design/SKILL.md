# Anti-AI Design Skill

Use this skill for landing pages, dashboards, SaaS interfaces, ecommerce pages and marketing websites.

## Goal
Avoid generic AI-generated visual patterns.

## Visual rules
- Do not create generic SaaS hero sections with excessive gradients.
- Do not overuse glassmorphism.
- Do not use repetitive 3-card grids in every section.
- Avoid random icons without semantic purpose.
- Avoid generic copy such as "transform your business with innovation".
- Create visual hierarchy with spacing, typography and contrast.
- Use asymmetry when appropriate.
- Use section rhythm: alternate dense, open, visual, proof, comparison and CTA sections.
- Prefer fewer, stronger components.
- Build from a clear design system.
- Every section must have a business purpose.

## Before coding
Evaluate:
1. What is the user supposed to understand?
2. What action should the user take?
3. What visual element makes this section memorable?
4. What can be removed?

## Output requirements
When proposing UI changes, include:
- Design rationale
- Components affected
- What was intentionally avoided
- Responsiveness notes
- Accessibility notes
