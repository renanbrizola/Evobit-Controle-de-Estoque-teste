# Project Guardian Skill

Use this skill before any meaningful change in this project.

## Purpose
Protect this existing project from unnecessary rewrites, generic AI design, unsafe refactors, token waste, and uncontrolled dependency growth.

## Mandatory rules
- Do not rewrite the project from scratch.
- Do not modify broad areas without an explicit plan.
- Do not install dependencies without justification.
- Do not read the entire codebase unless necessary.
- Do not change styling and business logic in the same step unless required.
- Do not create generic SaaS-like sections.
- Do not approve your own work without verification.
- Prefer small, reversible changes.
- Preserve existing working behavior unless the user explicitly asks to change it.

## Before editing
Always provide:
1. Task understanding
2. Files to inspect
3. Files likely to change
4. Risk level
5. Implementation plan
6. Acceptance criteria
7. What will not be changed

## After editing
Always provide:
1. Files changed
2. What changed
3. How to test
4. Remaining risks
5. Suggested next step
